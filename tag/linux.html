Raid 1+0 操作步骤记录

 2014-03-11 by Brant Xiong

操作结果

所有机器磁盘挂载情况如下：

[root@HOST work]# df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/sda2             8.2G  3.8G  4.4G  47% /
/dev/sda3             1.8T  1.7G  1.8T   1% /home
/dev/md10             8.9T  168M  8.4T   1% /home/data1

1. unmount正在使用的盘

for x in `seq 1 9`;do umount /disk$x;done
for x in `seq 10 11`;do umount /home/disk$x;done

2. raid 1方式做成5个2T的盘

i=0
for  x in bc de fg hi jk ;do
eval mdadm -Cv /dev/md$i -l 1 -n2 /dev/sd[${x}]1 <<EOF
    y
EOF
((i++))
sleep 3
done

3. raid 0方式做成10T的大盘

mdadm -Cv /dev/md10 -l 0 -n5 /dev/md[0-4] <<EOF
    y
EOF

4. 格式化md10为ext4

mkfs -t ext4 /dev/md10

5. 挂载大盘到/home/data1

mkdir /home/data1 && mount -t ext4 /dev/md10 /home/data1/

tcpdump用法简介
2014-02-18 by Brant Xiong

前言

今天在追查问题时，需要转包分析，用到了tcpdump工具，下面做下记录~

基本用法

tcpdump 采用命令行方式，需以root权限运行，它的命令格式为：

tcpdump [ -adeflnNOpqStvx ] [ -c 数量 ] [ -F 文件名 ]
        [ -i 网络接口 ] [ -r 文件名] [ -s snaplen ]
        [ -T 类型 ] [ -w 文件名 ] [表达式 ]

简单用法：

tcpdump -i eth1 port 8001
#查看localhost数据包
tcpdump -i lo port 8001

将结果写入文件：

tcpdump -i lo -nn port 8001 -w dump
